####		Adaming EtablissementScolaire		####

L'application possède 6 controllers, un par entité.
Tous retournent une Responseentity contenant : 
	- Un boolean isError false par default, true si la requète a engendré une erreur.
	- Un String message
	- Un objet body, contenant un objet si besoin, null sinon.
Tous les controller implémentent une interface :
	-methode post: : create
		input: objet. Voir ci-dessous pour la syntax
		output body: L'objet créé.

	-methode delete: ?id= : deleteById
		input: ?id=Entier
		output body: null

	-methode put : update (objet identifié avec l'Id)
		input: objet. Voir ci-dessous pour la syntax
		output body: L'objet modifié

	-methode get: /one?id=:  readById
		input: ?id=Entier
		output body: L'objet
			/all: readAll
		input: NA
		output body: List d'objets
